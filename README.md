# Refract - AI-Generated Code Cleanup

<system-reminder>
<parameter>Transform AI-generated code into maintainable, production-ready software.

Refract is a code analysis and refactoring pipeline that bridges the gap between rapid AI-assisted prototyping and production-grade software quality. It analyzes codebases generated by tools like Lovable, Bolt, and Cursor, identifies structural issues, and provides actionable suggestions to eliminate comprehension debt.
</parameter>
</system-reminder>

## Overview

AI coding tools have made it trivially fast to generate functional code. But speed comes at a cost: AI-generated codebases accumulate "comprehension debt" — code that works but is difficult to understand, maintain, or extend. Modules grow oversized, types devolve to `any`, dependencies tangle into cycles, and documentation falls behind.

Refract solves this. It is an AST-powered analysis engine that inspects your codebase, detects patterns of poor maintainability, and surfaces prioritized, actionable refactoring suggestions. For Pro and Team users, it can automate the entire refactoring workflow — generating pull requests with documented changes, updated documentation, and clean code structure — all while preserving the original behavior.

## Features

- **Code Analysis Engine** - AST-based parsing with 15+ built-in detectors
- **Issue System** - Prioritized suggestions with impact, effort, and blast radius calculations
- **Refactoring Suggestions** - Before/after code previews with batch acceptance
- **Auto-Refract (Pro/Team)** - Direct pull request generation with documentation
- **Documentation Generation** - Automatic README and API documentation
- **Code Quality Dashboard** - Health scores and trend tracking
- **Privacy-First** - All analysis runs client-side, no code sent to servers

## Getting Started

### Prerequisites

- Node.js 18+
- PostgreSQL 12+
- GitHub/GitLab account (for OAuth)

### Installation

#### Clone the Repository

```bash
git clone <repository-url>
cd refract
cd backend
npm install
cd ..
npm install
```

#### Backend Setup

1. **Configure backend environment**:
   ```bash
   cp backend/.env.example backend/.env
   ```

2. **Update backend environment variables**:
   ```bash
   # Set up PostgreSQL database
   # Create database: CREATE DATABASE refract;
   # Create user: CREATE USER refract_user WITH PASSWORD 'refract_password';
   # Grant privileges: GRANT ALL PRIVILEGES ON DATABASE refract TO refract_user;
   
   # Update backend/.env with your database URL
   # DATABASE_URL=postgresql://refract_user:refract_password@localhost:5432/refract
   ```

3. **Set up backend database**:
   ```bash
cd backend
npm run seed
```

#### Frontend Setup

1. **Configure frontend environment**:
   ```bash
   # Create .env file in the project root
   echo "VITE_API_URL=http://localhost:3001/api" > .env
   ```

2. **Install dependencies** (already done in step 1)

3. **Start development server**:
   ```bash
   npm run dev
   ```

#### Alternative: Docker Setup

For a quick setup, you can use Docker:

1. **Build and run Docker containers**:
   ```bash
   docker-compose up -d
   ```

2. **Check logs**:
   ```bash
   docker-compose logs -f
   ```

### Quick Start

Once the application is running:

1. **Open your browser** and navigate to `http://localhost:3000`
2. **Sign in** via GitHub or GitLab OAuth

## Features

### Code Analysis Engine

The Refract backend includes 15+ built-in detectors:

1. **Any type usage** - Detects TypeScript `any` type annotations
2. **Oversized components** - Identifies components exceeding size limits
3. **Dead state** - Finds unused state variables
4. **Missing documentation** - Detects undocumented components
5. **Console log leak** - Identifies debug console statements
6. **Effect dependencies** - Checks for missing React useEffect dependencies
7. **Prop drilling** - Detects excessive prop passing
8. **Generic naming** - Identifies non-descriptive variable/function names
9. **Circular dependencies** - Detects circular import dependencies
10. **State explosion** - Identifies components with too many state variables
11. **API in component** - Finds API calls directly in components
12. **Missing error boundary** - Detects missing error boundaries
13. **Memory leak** - Identifies potential memory leaks
14. **Duplicate logic** - Finds duplicate code patterns
15. **Unsafe cast** - Detects unsafe type casting

## Troubleshooting

### Common Issues

#### Database Connection Errors

If you encounter database connection errors:

1. **Check PostgreSQL is running**:
   ```bash
   sudo systemctl status postgresql
   ```

2. **Verify database exists**:
   ```bash
   psql -U postgres -l | grep refract
   ```

3. **Check connection string**:
   ```bash
   # Update .env file
   DATABASE_URL=postgresql://username:password@localhost:5432/refract
   ```

#### Authentication Issues

For OAuth authentication issues:

1. **Check client credentials**:
   ```bash
   # Verify client ID and secret in .env file
   GITHUB_CLIENT_ID=your-client-id
   GITHUB_CLIENT_SECRET=your-client-secret
   ```

2. **Verify redirect URIs**:
   ```
   # Should be configured in GitHub/GitLab application
   http://localhost:3000/api/auth/github/callback
   http://localhost:3001/api/auth/github/callback
   ```

For webhook issues:

1. **Verify webhook secrets**:
